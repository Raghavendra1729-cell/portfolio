# Overview

## Purpose
This portfolio is designed to present work and credentials clearly, with minimal visual distraction. The product goal is to keep attention on the engineer and projects, not UI effects.

## What it includes
- Public pages for professional profile, work history, projects, skills, and contact.
- Project details with links, stack context, and supporting visuals.
- Admin-only content management workflow.
- MongoDB-backed structured content models.

## Primary user journeys
1. **Recruiter / hiring manager**: Land on homepage, scan projects and experience, contact quickly.
2. **Engineer / collaborator**: Review project detail pages, stack decisions, and code/demo links.
3. **Owner/admin**: Sign in via admin secret, update content entries without changing code.
