# Content Editing

## Admin access
1. Open `/admin`.
2. Enter `ADMIN_SECRET`.
3. Edit collections from the dashboard tabs.

## Editable collections
- Projects
- Experience
- Education
- CP Profiles
- Hackathons
- Achievements
- Skills

## Content quality guidance
- Keep descriptions concise and outcome-oriented.
- Prefer measurable impact where possible.
- Keep links valid (`http://` or `https://`).
- Use project descriptions with this optional format for richer detail pages:
  - `Problem: ...`
  - `Approach: ...`
  - `Outcome: ...`

## Project entry recommendations
- `title`: short and specific.
- `description`: clear summary; optionally structured with Problem/Approach/Outcome lines.
- `techStack`: 4–10 relevant technologies.
- `link` and/or `repo`: primary external references.
- `images`: only useful screenshots.

## Ordering and visibility
Some collections support `order`, `featured`, or `isVisible`. Use these fields to control emphasis and display sequence.
